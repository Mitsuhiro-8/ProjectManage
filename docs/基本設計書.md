## 1. システム概要

### 1.1 目的

システム開発プロジェクトにおいて、メンバーの予定工数・実績工数と月間稼働カレンダーを可視化する。
プロジェクトマネージャーが工数計画と実績を一元管理し、リソース配分の把握を容易にする。

### 1.2 技術スタック

| 層 | 技術 | バージョン |
|---|---|---|
| フロントエンド | React + TypeScript + Vite | React 19 |
| UI ライブラリ | Material UI (MUI) | v7 |
| HTTP クライアント | axios | 最新安定版 |
| バックエンド | ASP.NET Core Web API (C#) | .NET 9 |
| ORM | Entity Framework Core | v9 |
| DB | SQLite | — |

### 1.3 動作環境

| 項目 | 内容 |
|---|---|
| バックエンド URL | http://localhost:5000 |
| フロントエンド URL | http://localhost:5173 |
| DB ファイル | `backend/projectmanage.db` |

## 6. アーキテクチャ設計

### 6.1 全体構成

```
ブラウザ
  │
  │ HTTP / JSON
  ▼
React (Vite) :5173
  ├── pages/          ← 画面単位のコンポーネント（useState + useEffect）
  ├── components/     ← 再利用可能な UI コンポーネント
  ├── api/            ← axios によるバックエンド API 呼び出し
  └── types/          ← TypeScript 型定義
  │
  │ HTTP / JSON (axios)
  │ ※ 開発時は Vite プロキシ経由
  ▼
ASP.NET Core Web API :5000
  ├── Controllers/    ← REST エンドポイント定義
  ├── Models/         ← EF Core エンティティ
  ├── Data/           ← DbContext + SeedData
  └── Program.cs      ← DI・ミドルウェア設定
  │
  │ EF Core
  ▼
SQLite (projectmanage.db)
```

### 6.2 バックエンド設計方針

| 方針 | 内容 |
|---|---|
| コントローラー粒度 | リソース単位（Projects / Members / ManHours / Holidays / Calendar） |
| DI パターン | primary constructor で AppDbContext を注入 |
| PUT 更新方式 | `FindAsync` → スカラー値のみ上書き（ナビゲーションプロパティ変更防止） |
| 削除方式 | Members: 論理削除（IsActive=false）/ その他: 物理削除 |
| N+1 防止 | 一覧取得は `Include` / バッチ処理は `ToHashSetAsync` 一括取得 |
| 入力バリデーション | コントローラー冒頭で範囲・重複チェックを実施し、不正は即 `BadRequest` |

### 6.3 フロントエンド設計方針

| 方針 | 内容 |
|---|---|
| 状態管理 | カスタムフック不使用。ページコンポーネント内で `useState` + `useEffect` を直接使用 |
| 再フェッチ | 外部トリガーによる再読み込みは `reloadKey` インクリメントパターンを使用 |
| 競合防止 | 非同期操作中は `loading` / `toggling` フラグで UI を `disabled` にする |
| 型安全 | `types/index.ts` に全エンティティの TypeScript インターフェースを集約 |

### 6.4 祝日自動計算ロジック

`JapanHolidayCalculator.Calculate(year)` で以下を計算する:

| 区分 | 計算方法 |
|---|---|
| 固定祝日 | 元日・建国記念日・天皇誕生日・昭和の日・憲法記念日・みどりの日・こどもの日・山の日・文化の日・勤労感謝の日（10 件） |
| ハッピーマンデー | 成人の日（1月第2月曜）・海の日（7月第3月曜）・敬老の日（9月第3月曜）・スポーツの日（10月第2月曜） |
| 春分・秋分の日 | 国立天文台式の近似計算式（1980〜2099年対応） |
| 振替休日 | 祝日が日曜の場合、翌月曜以降の平日（既存祝日と重複しない日）に振替 |

## 7. 非機能要件

### 7.1 セキュリティ

| 項目 | 対応内容 |
|---|---|
| SQL インジェクション | EF Core のパラメータ化クエリにより防止 |
| XSS | React の JSX エスケープ + MUI コンポーネントにより防止 |
| CORS | 開発環境では `http://localhost:5173` のみ許可 |
| 認証・認可 | 現バージョンでは未実装（ローカル内部利用前提） |

### 7.2 データ整合性

| 項目 | 対応内容 |
|---|---|
| 工数の重複 | (ProjectId, MemberId, Year, Month) のユニーク制約 + UPSERT で対応 |
| メンバーアサイン重複 | (ProjectId, MemberId) のユニーク制約 + 事前 `AnyAsync` チェックで 409 を返す |
| 祝日重複 | Date のユニーク制約 + シード時は HashSet で重複スキップ |
| メンバー削除 | 論理削除のみ（IsActive=false）。工数・アサイン履歴は保持する |

### 7.3 起動手順

```bash
# 一括起動
./start.sh

# 個別起動
cd backend && dotnet run      # http://localhost:5000
cd frontend && npm run dev    # http://localhost:5173
```

初回起動時にデータベース（`backend/projectmanage.db`）が自動作成され、デモデータが投入される。
