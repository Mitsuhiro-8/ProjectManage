## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 機能名 | メンバー管理 |
| 目的 | プロジェクトにアサインするメンバーの登録・編集・削除を行う |
| 対象ユーザー | プロジェクトマネージャー |

### 機能一覧

| # | 機能 | 説明 |
|---|---|---|
| 2-1 | メンバー一覧表示 | アクティブなメンバーを氏名昇順で表示 |
| 2-2 | メンバー作成 | 新規メンバーを登録する |
| 2-3 | メンバー編集 | メンバー情報を更新する |
| 2-4 | メンバー削除 | メンバーを論理削除する（IsActive = false） |

### 削除仕様（論理削除）

メンバーを削除しても、過去の工数データ・アサイン履歴は保持される。
削除後はメンバー一覧から非表示になり、新規アサインもできなくなる。

## 3. API設計

### GET /api/members

**概要:** アクティブなメンバー一覧取得（IsActive=true、Name 昇順）

**レスポンス例:**
```json
[
  {
    "id": 1,
    "name": "山田 太郎",
    "email": "yamada@example.com",
    "role": "PM",
    "defaultMonthlyHours": 160,
    "isActive": true,
    "createdAt": "2026-01-01T00:00:00Z",
    "updatedAt": "2026-01-01T00:00:00Z"
  }
]
```

### POST /api/members

**概要:** メンバー作成

**リクエストボディ:**
```json
{
  "name": "山田 太郎",
  "email": "yamada@example.com",
  "role": "PM",
  "defaultMonthlyHours": 160
}
```

**レスポンス:** 201 Created + 作成したメンバーオブジェクト

### PUT /api/members/{id}

**概要:** メンバー更新（スカラー値のみ）

**リクエストボディ:** POST と同形式（id フィールドを含む）

**レスポンス:** 204 No Content / 400 Bad Request / 404 Not Found

### DELETE /api/members/{id}

**概要:** メンバー論理削除（IsActive = false）

**レスポンス:** 204 No Content / 404 Not Found

## 4. データフロー

### 4.1 メンバー作成フロー

```
[ユーザー] メンバー追加ボタンクリック
    → MemberFormDialog を開く（editMember = null）
    → フォーム入力
    → 保存ボタンクリック
    → memberApi.create(formData)
    → POST /api/members
    → 201 Created
    → ダイアログを閉じる
    → memberApi.getAll() で一覧を再取得
```

### 4.2 メンバー削除フロー

```
[ユーザー] 削除アイコンクリック
    → window.confirm で確認
    → 確認OK
    → memberApi.delete(id)
    → DELETE /api/members/{id}
    → 204 No Content
    → memberApi.getAll() で一覧を再取得
    ※ バックエンドは IsActive = false に更新（物理削除ではない）
```

## 5. バリデーション

### バックエンド

| フィールド | ルール | エラーレスポンス |
|---|---|---|
| id（PUT） | URL の id とボディの id が一致すること | 400 Bad Request |
| id（PUT/DELETE） | 該当レコードが存在すること | 404 Not Found |

### フロントエンド

| フィールド | ルール |
|---|---|
| 氏名 | 空文字不可 |
| 月間標準工数 | 必須、正の数値 |
