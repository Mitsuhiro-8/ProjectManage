# タスク管理機能 設計書

**作成日:** 2026-03-29
**対象機能:** タスク管理

---

## 1. 機能概要

プロジェクト・担当者（メンバー）に紐付くタスクを管理し、タスク単位での予定工数・実績工数を記録する。
プロジェクト別・メンバー別に工数の予実を集計して可視化する。

---

## 2. 機能要件

| # | 機能 | 説明 |
|---|---|---|
| 1 | タスク一覧表示 | プロジェクト・担当者・ステータスでフィルタリングしてテーブル表示 |
| 2 | タスク作成 | プロジェクト・担当者・工数情報を入力して登録 |
| 3 | タスク編集 | タスク情報・工数・ステータスを更新 |
| 4 | タスク削除 | タスクの物理削除 |
| 5 | 工数集計 | プロジェクト別・メンバー別に予定工数・実績工数の合計を表示 |

## 4. API設計

### 4.1 エンドポイント一覧

| メソッド | パス | クエリパラメータ | 説明 | 成功レスポンス |
|---|---|---|---|---|
| GET | /api/tasks | projectId?, memberId?, status? | タスク一覧取得 | 200 + Task[] |
| GET | /api/tasks/{id} | — | タスク取得 | 200 + Task / 404 |
| POST | /api/tasks | — | タスク作成 | 201 + Task |
| PUT | /api/tasks/{id} | — | タスク更新 | 204 / 400 / 404 |
| DELETE | /api/tasks/{id} | — | タスク削除 | 204 / 404 |
| GET | /api/tasks/summary | projectId?, memberId? | 工数集計取得 | 200 + TaskSummary[] |

### 4.2 Task レスポンス構造

```json
{
  "id": 1,
  "projectId": 1,
  "memberId": 2,
  "name": "基本設計書作成",
  "description": "システム全体の基本設計を行う",
  "status": "InProgress",
  "plannedHours": 16.0,
  "actualHours": 10.0,
  "startDate": "2026-04-01",
  "endDate": "2026-04-15",
  "createdAt": "2026-03-29T00:00:00Z",
  "updatedAt": "2026-03-29T00:00:00Z",
  "project": { "id": 1, "name": "プロジェクトA" },
  "member": { "id": 2, "name": "山田 太郎" }
}
```

### 4.3 TaskSummary レスポンス構造

集計キー（projectId or memberId）ごとの工数合計。

```json
[
  {
    "projectId": 1,
    "projectName": "プロジェクトA",
    "memberId": 2,
    "memberName": "山田 太郎",
    "taskCount": 5,
    "plannedHours": 80.0,
    "actualHours": 60.0
  }
]
```

### 4.4 入力バリデーション

| 項目 | ルール |
|---|---|
| Name | 必須、1文字以上 |
| ProjectId | 必須、存在するProjectのId |
| MemberId | 必須、存在するMemberのId（IsActive=true） |
| PlannedHours | 必須、0以上 |
| ActualHours | 任意、0以上（入力時） |
| Status | Todo / InProgress / Done のいずれか |

## 6. バックエンド実装方針

### 6.1 モデル

`backend/Models/Task.cs` を新規作成。

```csharp
public class Task
{
    public int Id { get; set; }
    public int ProjectId { get; set; }
    public int MemberId { get; set; }
    public string Name { get; set; } = string.Empty;
    public string? Description { get; set; }
    public TaskStatus Status { get; set; } = TaskStatus.Todo;
    public double PlannedHours { get; set; } = 0;
    public double? ActualHours { get; set; }
    public DateOnly? StartDate { get; set; }
    public DateOnly? EndDate { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }

    public Project Project { get; set; } = null!;
    public Member Member { get; set; } = null!;
}

public enum TaskStatus { Todo = 0, InProgress = 1, Done = 2 }
```

### 6.2 コントローラー（TasksController）

- `GET /api/tasks` : projectId / memberId / status でフィルタ、Project・Member を Include
- `GET /api/tasks/summary` : projectId or memberId でグループ集計（GroupBy → Select で集約）
  - ※ `/summary` は `{id}` より先にルーティングが当たるよう `[HttpGet("summary")]` を先に定義する
- PUT: `FindAsync` → スカラー値のみ上書き（規約準拠）
- ループ内DBクエリ禁止

### 6.3 AppDbContext への追加

`public DbSet<Task> Tasks { get; set; }` を追加。

### 6.4 EFマイグレーション

本プロジェクトは `EnsureCreated` を使用しているため、DB ファイルを削除して再起動することでテーブルを再作成する。

## 8. テスト計画

| テストケース |
|---|
| GET /tasks : フィルタなしで全件取得できる |

| GET /tasks?memberId=1 : 指定メンバーのタスクのみ返る |
| GET /tasks?status=Todo : 指定ステータスのタスクのみ返る |
| GET /tasks/summary : 集計結果が正しく返る |
| POST /tasks : タスクが作成されCreatedAtが設定される |
