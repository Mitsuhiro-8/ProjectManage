## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 機能名 | プロジェクト管理 |
| 目的 | プロジェクトの登録・編集・削除・メンバーアサインを行う |
| 対象ユーザー | プロジェクトマネージャー |

### 機能一覧

| # | 機能 | 説明 |
|---|---|---|
| 1-1 | プロジェクト一覧表示 | 全プロジェクトを開始日昇順で表示 |
| 1-2 | プロジェクト作成 | 新規プロジェクトを登録する |
| 1-3 | プロジェクト編集 | プロジェクト情報を更新する |
| 1-4 | プロジェクト削除 | プロジェクトを物理削除する |
| 1-5 | メンバーアサイン | プロジェクトにメンバーを追加・解除する |

## 3. API設計

### GET /api/projects

**概要:** プロジェクト一覧取得

**レスポンス例:**
```json
[
  {
    "id": 1,
    "name": "○○システム開発",
    "description": "新規開発案件",
    "startDate": "2026-01-01",
    "endDate": "2026-06-30",
    "status": "Active",
    "createdAt": "2026-01-01T00:00:00Z",
    "updatedAt": "2026-01-01T00:00:00Z",
    "projectMembers": [
      {
        "id": 1,
        "projectId": 1,
        "memberId": 1,
        "assignedAt": "2026-01-01T00:00:00Z",
        "member": { "id": 1, "name": "山田 太郎", "role": "PM" }
      }
    ]
  }
]
```

### POST /api/projects

**概要:** プロジェクト作成

**リクエストボディ:**
```json
{
  "name": "○○システム開発",
  "description": "説明",
  "startDate": "2026-01-01",
  "endDate": "2026-06-30",
  "status": "Active"
}
```

**レスポンス:** 201 Created + 作成したプロジェクトオブジェクト

### PUT /api/projects/{id}

**概要:** プロジェクト更新（スカラー値のみ）

**リクエストボディ:** POST と同形式（id フィールドを含む）

**レスポンス:** 204 No Content / 400 Bad Request / 404 Not Found

### DELETE /api/projects/{id}

**概要:** プロジェクト物理削除

**レスポンス:** 204 No Content / 404 Not Found

### POST /api/projects/{id}/members/{memberId}

**概要:** メンバーをプロジェクトにアサイン

**レスポンス:** 200 OK / 409 Conflict（既にアサイン済み）

### DELETE /api/projects/{id}/members/{memberId}

**概要:** メンバーのアサインを解除

**レスポンス:** 204 No Content / 404 Not Found

## 4. データフロー

### 4.1 プロジェクト作成フロー

```
[ユーザー] 新規作成ボタンクリック
    → ProjectFormDialog を開く（editProject = null）
    → フォーム入力
    → 保存ボタンクリック
    → projectApi.create(formData)
    → POST /api/projects
    → 201 Created
    → ダイアログを閉じる
    → projectApi.getAll() で一覧を再取得
```

### 4.2 メンバーアサインフロー

```
[ユーザー] アサインアイコンクリック
    → MemberAssignDialog を開く（対象プロジェクトを渡す）
    → memberApi.getAll() で全メンバー取得
    → project.projectMembers からアサイン済み ID を初期化
    → チェックボックス操作
        [オンにした場合] projectApi.assignMember(projectId, memberId) → POST
        [オフにした場合] projectApi.removeMember(projectId, memberId) → DELETE
    → onUpdated() で親コンポーネントの一覧を再取得
```

## 5. バリデーション

### バックエンド

| フィールド | ルール | エラーレスポンス |
|---|---|---|
| id（PUT） | URL の id とボディの id が一致すること | 400 Bad Request |
| id（PUT/DELETE） | 該当レコードが存在すること | 404 Not Found |
| アサイン（POST） | 同一 ProjectId + MemberId が存在しないこと | 409 Conflict |

### フロントエンド

| フィールド | ルール |
|---|---|
| プロジェクト名 | 空文字不可 |
| 開始日 | 必須 |
| 終了日 | 必須、開始日以降 |
