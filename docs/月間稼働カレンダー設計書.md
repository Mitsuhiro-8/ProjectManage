## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 機能名 | 月間稼働カレンダー |
| 目的 | 月別・メンバー別の稼働予定と実績工数を視覚的に把握する |
| 対象ユーザー | プロジェクトマネージャー |

### 機能一覧

| # | 機能 | 説明 |
|---|---|---|
| 4-1 | カレンダー表示 | 年月・プロジェクトでフィルタしてカレンダーを表示 |
| 4-2 | 月ナビゲーション | 前月・次月に切り替える |
| 4-3 | 稼働日カウント | 土日・祝日を除いた稼働日数を表示 |
| 4-4 | 祝日シード | 指定年の日本の祝日を自動計算して DB に登録する |

## 3. API設計

### GET /api/calendar

**概要:** 指定年月のカレンダーデータ取得

**クエリパラメータ:**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| year | int | ○ | 対象年 |
| month | int | ○ | 対象月（1〜12） |
| projectId | int | — | プロジェクットIDでフィルタ（未指定時は全プロジェクト） |

**レスポンス例:**
```json
{
  "year": 2026,
  "month": 3,
  "workingDays": 20,
  "holidays": [
    "2026-03-20"
  ],
  "members": [
    {
      "memberId": 1,
      "memberName": "山田 太郎",
      "dailyPlannedHours": 8.0,
      "plannedHours": 160.0,
      "actualHours": 140.0
    },
    {
      "memberId": 2,
      "memberName": "鈴木 花子",
      "dailyPlannedHours": 6.0,
      "plannedHours": 120.0,
      "actualHours": null
    }
  ]
}
```

### POST /api/holidays/seed/{year}

**概要:** 指定年の日本の祝日を自動計算して DB に登録

**パスパラメータ:**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| year | int | ○ | 対象年（1980〜2099） |

**レスポンス:**
- 200 OK: `{ "added": N }` — 新たに追加した件数
- 400 Bad Request: 年が 1980〜2099 の範囲外

## 4. データフロー

### 4.1 カレンダー表示フロー

```
[ユーザー] カレンダー画面を開く、または年月・プロジェクットを変更
    → calendarApi.get({ year, month, projectId })
    → GET /api/calendar?year=&month=&projectId=
    → CalendarData を受け取る
    → 稼働日数 Chip を更新
    → カレンダーテーブルを再描画
```

### 4.2 祝日シードフロー

```
[ユーザー] 「{year}年の祝日をシード」ボタンクリック
    → setSeeding(true)（ボタン disabled）
    → holidayApi.seed(year)
    → POST /api/holidays/seed/{year}
    → { added: N } を受け取る
    → setSeeding(false)
    → setReloadKey(k => k + 1) でカレンダーを再取得
    ※ reloadKey を useEffect の依存配列に含めることで再フェッチをトリガーする
```

### 4.3 月ナビゲーションフロー

```
[ユーザー] ◀ または ▶ ボタンクリック
    → year / month を更新（1月→12月 または 12月→1月 の年またぎも考慮）
    → useEffect が year / month の変化を検知
    → calendarApi.get を再呼び出し
```

## 5. 稼働日・工数計算仕様

### 5.1 稼働日の定義

以下をすべて除いた日を稼働日とする:

- 土曜日（DayOfWeek.Saturday）
- 日曜日（DayOfWeek.Sunday）
- `Holidays` テーブルに登録された日付

### 5.2 日次予定工数の算出

```
dailyPlannedHours = ManHour.PlannedHours / 稼働日数
```

メンバーごとに当月の工数レコード（ManHour）から算出する。

### 5.3 月間予定工数・実績工数

- `plannedHours` = ManHour.PlannedHours の合計
- `actualHours` = ManHour.ActualHours の合計（null は除外。全件 null の場合は null を返す）

## 6. 祝日自動計算仕様

`JapanHolidayCalculator.Calculate(year)` で以下を自動計算する（対象年: 1980〜2099）:

### 6.1 固定祝日

| 祝日名 | 月日 |
|---|---|
| 元日 | 1月1日 |
| 建国記念の日 | 2月11日 |
| 天皇誕生日 | 2月23日 |
| 昭和の日 | 4月29日 |
| 憲法記念日 | 5月3日 |
| みどりの日 | 5月4日 |
| こどもの日 | 5月5日 |
| 山の日 | 8月11日 |
| 文化の日 | 11月3日 |
| 勤労感謝の日 | 11月23日 |

### 6.2 ハッピーマンデー祝日

| 祝日名 | 計算方法 |
|---|---|
| 成人の日 | 1月第2月曜日 |
| 海の日 | 7月第3月曜日 |
| 敬老の日 | 9月第3月曜日 |
| スポーツの日 | 10月第2月曜日 |

### 6.3 移動祝日

| 祝日名 | 計算方法 |
|---|---|
| 春分の日 | `floor(20.8431 + 0.242194 * (year - 1980) - floor((year - 1980) / 4.0))`（3月） |
| 秋分の日 | `floor(23.2488 + 0.242194 * (year - 1980) - floor((year - 1980) / 4.0))`（9月） |

### 6.4 振替休日

祝日が日曜日に当たる場合、翌月曜日以降の最初の平日（既存祝日と重複しない日）を振替休日とする。
