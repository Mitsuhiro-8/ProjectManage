## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 機能名 | 工数管理 |
| 目的 | プロジェクト・メンバーごとの月次予定工数・実績工数を登録・管理する |
| 対象ユーザー | プロジェクトマネージャー・各メンバー |

### 機能一覧

| # | 機能 | 説明 |
|---|---|---|
| 3-1 | 工数一覧表示 | 月次工数データをプロジェクト・年月でフィルタして表示 |
| 3-2 | 工数入力 | 予定工数・実績工数を新規登録（同一キーが存在する場合は更新） |
| 3-3 | 工数編集 | 登録済み工数データを更新する |
| 3-4 | 工数削除 | 工数データを物理削除する |

### UPSERT 仕様

`POST /api/manhours` は同一の (ProjectId, MemberId, Year, Month) が存在する場合、
新規作成ではなく既存レコードを更新する（UPSERT）。

## 3. API設計

### GET /api/manhours

**概要:** 工数一覧取得（フィルタ可能）

**クエリパラメータ:**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| projectId | int | — | プロジェクトIDでフィルタ |
| year | int | — | 年でフィルタ |
| month | int | — | 月でフィルタ |

**レスポンス例:**
```json
[
  {
    "id": 1,
    "projectId": 1,
    "memberId": 1,
    "year": 2026,
    "month": 3,
    "plannedHours": 160,
    "actualHours": 140,
    "memo": null,
    "updatedAt": "2026-03-01T00:00:00Z",
    "project": { "id": 1, "name": "○○システム開発" },
    "member": { "id": 1, "name": "山田 太郎" }
  }
]
```

### POST /api/manhours（UPSERT）

**概要:** 工数登録（同一キーが存在すれば更新）

**リクエストボディ:**
```json
{
  "projectId": 1,
  "memberId": 1,
  "year": 2026,
  "month": 3,
  "plannedHours": 160,
  "actualHours": 140,
  "memo": "備考テキスト"
}
```

**レスポンス:**
- 新規作成: 201 Created + 作成データ
- 既存更新: 200 OK + 更新データ

### PUT /api/manhours/{id}

**概要:** 工数更新（スカラー値のみ）

**リクエストボディ:** POST と同形式（id フィールドを含む）

**レスポンス:** 204 No Content / 400 Bad Request / 404 Not Found

### DELETE /api/manhours/{id}

**概要:** 工数物理削除

**レスポンス:** 204 No Content / 404 Not Found

### GET /api/manhours/summary

**概要:** 指定メンバーの年間月次工数サマリー取得

**クエリパラメータ:**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| memberId | int | ○ | メンバーID |
| year | int | ○ | 対象年 |

**レスポンス:** 200 OK + ManHour[]（月昇順）

## 4. データフロー

### 4.1 工数入力フロー（新規）

```
[ユーザー] 工数入力ボタンクリック
    → ManHourInputDialog を開く（editManHour = null）
    → フォーム入力（プロジェクト・メンバー・年月・予定工数等）
    → 保存ボタンクリック
    → manHourApi.create(formData)
    → POST /api/manhours
        ┌ 同一キーなし → 201 Created（新規登録）
        └ 同一キーあり → 200 OK（既存更新）
    → ダイアログを閉じる
    → 工数一覧を再取得
```

### 4.2 工数編集フロー

```
[ユーザー] 編集アイコンクリック
    → ManHourInputDialog を開く（editManHour = 対象データ）
    → フォームに既存データをプリセット
    → 値を変更して保存
    → manHourApi.update(id, formData)
    → PUT /api/manhours/{id}
    → 204 No Content
    → ダイアログを閉じる
    → 工数一覧を再取得
```
